# 问题库使用说明（GEO 时代的"关键词"）

> 实际数据在 `../tool/data/queries.csv`（工具层和工作流层共用同一份，别建两份）。
> 本文档说明怎么维护它。映射 CSDN 文章「关键词语义分析」模块的输入端。

## 这是什么

问题库 = 预测用户在 AI 引擎里会怎么问这家工厂。这就是 GEO 时代的"关键词"。
后面一切监测和写作都以它为靶子。

## 字段说明（queries.csv）

| 字段 | 含义 | 取值 |
|---|---|---|
| id | 问题编号 | Q01、Q02……（hot_keywords.py 自动续号） |
| query | 问题原文 | 用户在 AI 里真实会打的那句话 |
| intent | 意图分类 | 选型/对比/参数/资质/趋势/口碑 |
| priority_seed | 业务价值种子 | 1–5，5=最想拿下的高价值问题 |
| status | 状态 | active=参与排序 / parked=暂时搁置 |

## 六类意图（化妆品代工样例）

| 意图 | 含义 | 样例 |
|---|---|---|
| 选型 | 用户在找代工厂 | 化妆品代工厂怎么选、面膜代工厂哪家好 |
| 对比 | 用户在做选择题 | OEM和ODM哪个适合小品牌、广州vs上海代工 |
| 参数 | 用户问具体条件 | MOQ最低多少、打样周期多久 |
| 资质 | 用户在评估可信度 | 代工需要什么资质、怎么判断资质靠不靠谱 |
| 趋势 | 用户问行业方向 | 2026护肤代工趋势、功效护肤代工 |
| 口碑 | 用户查具体工厂 | XX工厂代工怎么样、避坑指南 |

## 怎么维护

1. **初始**：每类意图先列 5–10 条，按"客户真会问 + 你有竞争力答"筛到 20–30 条。
2. **扩充**：跑 `python tool/hot_keywords.py` 把行业热词自动派生成新问题入库。
3. **调权**：每月复盘后，根据实际询盘价值调 priority_seed。
4. **搁置**：暂时不做的问题设 status=parked，不参与排序但不丢失。

## 🔧 换行业怎么改

把 query 列换成你那个行业用户会问的话，intent 六分类通常通用，
priority_seed 按你的业务重心重设。改完跑 geo_engine.py 即可。
